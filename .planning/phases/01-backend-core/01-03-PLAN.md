**Step 0 — Create .env with real credentials (do this first):**
```
# File: cyber-god/backend/.env
# DO NOT commit this file to git
ZHIPU_API_KEY=your_actual_zhipu_api_key
GLM_MODEL=glm-4-flash
ALLOWED_ORIGINS=http://localhost:3000
```

**Step 1 — Start the server (new terminal, keep running):**
```powershell
cd cyber-god\backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
Expected: `INFO: Application startup complete.` (no errors)

**Step 2 — Test CORS preflight (new terminal):**
```powershell
curl -s -X OPTIONS http://localhost:8000/api/chat `
  -H "Origin: http://localhost:3000" `
  -H "Access-Control-Request-Method: POST" `
  -v 2>&1 | Select-String "access-control|200"
```
Expected: Response includes `access-control-allow-origin: http://localhost:3000` and `200 OK`

**Step 3 — Test streaming chat (the main Phase 1 gate):**
```powershell
curl -N -s -X POST http://localhost:8000/api/chat `
  -H "Content-Type: application/json" `
  -d '{"messages":[{"role":"user","content":"我想花800块买个盲盒"}],"savings":5000,"target":10000}'
```
Expected output (streams line by line):
```
f:{"id":"..."}
0:"【驳回】"   <- or 【批准】
0:"..."        <- more Chinese text tokens
...
2:[{"new_savings":...,"progress_pct":...,"delta":-...}]
e:{"finishReason":"stop",...}
d:{"finishReason":"stop",...}
```

**Step 4 — Verify x-vercel-ai-data-stream header:**
```powershell
curl -s -D - -X POST http://localhost:8000/api/chat `
  -H "Content-Type: application/json" `
  -d '{"messages":[{"role":"user","content":"我想花800块买个盲盒"}],"savings":5000,"target":10000}' `
  | Select-String "x-vercel-ai-data-stream"
```
Expected: `x-vercel-ai-data-stream: v1`

**Step 5 — Check server logs for GLM model confirmation:**
In the server terminal, verify there are no `model not found` or `authentication failed` errors after Step 3.
