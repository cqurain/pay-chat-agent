  <action>
In `cyber-god/backend/main.py`, inside the `lifespan()` async context manager, change the `StdioServerParameters` call so that `command="python3"` instead of `command="python"`.

The `python:3.12-slim` Docker base image only places `python3` in PATH by default. If `command="python"` is left unchanged, the MCP server subprocess will fail to launch inside the container with a `FileNotFoundError`.

Before change:
```python
server_params = StdioServerParameters(
    command="python",
    args=[MCP_SERVER_PATH],
    env=None,
)
```

After change:
```python
server_params = StdioServerParameters(
    command="python3",
    args=[MCP_SERVER_PATH],
    env=None,
)
```

This is a one-line change. Leave all other lines in main.py exactly as they are.

Note: `python3` also works in the local dev environment (Windows `py` launcher aside — but in local dev the virtual environment's `python` is in PATH through `.venv/Scripts/python3.exe`). If local dev breaks with `python3`, the fallback is to add `RUN ln -s /usr/local/bin/python3 /usr/local/bin/python` in the Dockerfile instead — but try `python3` first as it is cleaner.
  </action>
  <verify>
    `grep "command=\"python3\"" cyber-god/backend/main.py` returns a match.
    `grep "command=\"python\"" cyber-god/backend/main.py` returns NO match (old value removed).
  </verify>
  <done>
    main.py lifespan() uses `command="python3"` in StdioServerParameters. The MCP subprocess will correctly resolve in the python:3.12-slim container.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker host → container | env_file passes secrets from host filesystem into container env |
| Container → Zhipu API | ZHIPU_API_KEY leaves the host via HTTPS to open.bigmodel.cn |
| Browser → Docker host port 8000 | Unauthenticated HTTP(S) access to the FastAPI endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Dockerfile COPY . . | mitigate | .dockerignore excludes .env and .env.* — secrets never enter image layers; verified by `docker build --dry-run` or `docker history` check |
| T-03-02 | Information Disclosure | .env.example committed to repo | accept | .env.example contains only placeholder values (sk-your-zhipu-api-key-here); real key stays in .env which is gitignored |
| T-03-03 | Spoofing | CORS ALLOWED_ORIGINS | mitigate | config.py already parses ALLOWED_ORIGINS as explicit list; .env.example defaults to http://localhost:3000 only; README instructs user to set exact Vercel URL for production |
| T-03-04 | Denial of Service | MCP stdio subprocess hang | mitigate | PYTHONUNBUFFERED=1 set in Dockerfile ENV — prevents Python block buffering in non-TTY mode that causes indefinite hang |
| T-03-05 | Elevation of Privilege | Container running as root | accept | PoC scope; python:3.12-slim defaults to root inside container; adding a non-root USER is a v2 hardening task |
</threat_model>
