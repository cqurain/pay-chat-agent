<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → localStorage | User-controlled; any JS running on the page can read/write savings and target values |
| browser → FastAPI backend | All user-supplied text (purchase impulse) is sent to the backend; backend must sanitize |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .env.local | mitigate | .env.local must be gitignored (NEXT_PUBLIC_ prefix is acceptable for non-secret backend URL; no secrets stored in frontend env) |
| T-02-02 | Tampering | localStorage (gsd_savings, gsd_target) | accept | Values are user-controlled input by design; no trust boundary crossed — frontend only reads them back and sends to its own backend |
| T-02-03 | Information Disclosure | NEXT_PUBLIC_API_URL | accept | Intentionally public — it is the backend URL, not a secret. NEXT_PUBLIC_ prefix is correct. |
| T-02-04 | Spoofing | lib/types.ts SavingsPayload | mitigate | useEffect that reads useChat.data must validate payload shape before calling setSavings (check new_savings is number, not NaN) — implemented in Plan 03 |
</threat_model>
