# Quick 260420-v98: Refactor Cyber God Backend Agent to ReAct-lite Summary

**One-liner:** Replaced fragile _extract_intent + synthetic tool injection pipeline with a real ReAct-lite loop where GLM-5 natively drives search_product_price tool calls, with MAX_TOOL_ROUNDS=5 guard, per-tool 12s timeout with 2x exponential backoff, and multi-item savings aggregation.

## What Was Built

### Task 1 — ReAct-lite Agent Loop (loop.py + savings.py)

Completely rewrote `loop.py`. Removed the four-step pre-extraction pipeline:
- `_extract_intent` (glm-4-flash JSON mode to find product + stated price)
- `_resolve_price` (user_stated skip / MCP call branch)
- `_build_price_context` + `_build_tool_context` + `_inject_tool_exchange` (synthetic tool message injection)
- `_build_no_intent_sentinel`, `_build_savings_payload` (replaced by `_build_multi_savings_payload`)

Added:
- `TOOLS` module-level constant — one entry: `search_product_price`
- `_execute_tool(name, args, mcp_session)` — dispatches by name; sanitizes `product_name[:100]` (T-v98-01); returns `{"error": "unknown tool: X"}` for unrecognized names (T-v98-04)
- `_execute_tool_with_retry(tc, mcp_session, retries=2)` — `asyncio.wait_for(timeout=12.0)` with `0.5 * 2**attempt` backoff (T-v98-03)
- `_run_agentic_loop(messages, glm_client, model, mcp_session)` — loops up to `MAX_TOOL_ROUNDS=5` (T-v98-02); calls GLM non-streaming with `tools=TOOLS`; appends assistant + tool result messages; breaks when no `tool_calls` returned
- `_extract_price_results(messages)` — scans tool messages, skips entries with `"error"` key (T-v98-05), returns valid price dicts
- `_build_multi_savings_payload(price_results, savings, target, impact)` — handles empty/single/multi-item cases; includes `items[]` when len > 1

Updated `run_agent_loop` body to: emit `f:`, compress history if needed, call `_run_agentic_loop`, aggregate total_price, calculate impact, inject tx+savings context as system message, emit `2:` payload, stream verdict.

Added `calculate_combined_impact` to `tools/savings.py` for external multi-item aggregation.

### Task 2 — Prompt, Types, PriceResearchCard (prompt.py, types.ts, PriceResearchCard.tsx)

**prompt.py:** Replaced "系统已把所有数字算好塞给你了，你只负责回应" header in both `_DATA_RULES_SNARKY` and `_DATA_RULES_GENTLE` with tool-calling instructions explaining `search_product_price` usage. All other rules (confidence handling, consecutive_spend_days logic, verdict format, style examples) preserved unchanged.

**types.ts:** Added `items?: Array<{ name: string; price: number; confidence: string }>` to `SavingsPayload`.

**PriceResearchCard.tsx:** Destructures `items` from payload. Renders a `space-y-0.5` list of item rows (truncated name + price) after the sourceTag block when `items && items.length > 1`. Single-item and no-intent layouts unchanged (backward compatible).
