2. `async def _execute_tool(name: str, args: dict, mcp_session: ClientSession) -> str`:
   - If `name == "search_product_price"`: sanitize `product_name = str(args.get("product_name", "")).strip()[:100]`, call `await get_price(product_name, mcp_session)`, return `json.dumps(result, ensure_ascii=False)`
   - Else: return `json.dumps({"error": f"unknown tool: {name}"}, ensure_ascii=False)`

3. `async def _execute_tool_with_retry(tc: dict, mcp_session: ClientSession, retries: int = 2) -> tuple[str, str]`:
   - Extract `tc_id = tc["id"]`, `name = tc["function"]["name"]`, `args = json.loads(tc["function"]["arguments"] or "{}")`
   - Loop up to `retries + 1` times with `asyncio.wait_for(..., timeout=12.0)`
   - On `TimeoutError` or `Exception`: wait `0.5 * 2**attempt` seconds then retry
   - After all retries exhausted: return `(tc_id, json.dumps({"error": "timeout"}, ensure_ascii=False))`
   - On success: return `(tc_id, result_str)`

4. `async def _run_agentic_loop(messages: list[dict], glm_client: AsyncOpenAI, model: str, mcp_session: ClientSession) -> tuple[list[dict], list[dict]]`:
   - `MAX_TOOL_ROUNDS = 5`
   - Loop up to MAX_TOOL_ROUNDS:
     - Call GLM non-streaming: `resp = await glm_client.chat.completions.create(model=model, messages=messages, tools=TOOLS, tool_choice="auto", stream=False)`
     - `msg = resp.choices[0].message`
     - If `msg.tool_calls` is None or empty: break (GLM done)
     - Append assistant message: `{"role": "assistant", "content": msg.content, "tool_calls": [tc.model_dump() for tc in msg.tool_calls]}`
     - Execute all tool calls in parallel: `results = await asyncio.gather(*[_execute_tool_with_retry(tc.model_dump(), mcp_session) for tc in msg.tool_calls])`
     - Append one tool result message per result: `{"role": "tool", "tool_call_id": tc_id, "content": result_str}`
   - Return `(messages, price_results)` where `price_results` is extracted by `_extract_price_results(messages)`

5. `def _extract_price_results(messages: list[dict]) -> list[dict]`:
   - Scan messages for `role == "tool"` entries
   - Parse each content as JSON; skip if contains `"error"` key
   - Return list of valid price dicts (those that have `"price"` key)

6. `def _build_multi_savings_payload(price_results: list[dict], impact: dict) -> dict`:
   - Build the 2: payload. If `price_results` is empty: emit sentinel with `confidence="no_intent"`, `delta=0`, `new_savings=savings`, `progress_pct=progress`.
   - If single item: same shape as old `_build_savings_payload` (backward compat).
   - If multiple items: include `items: [{"name": r["name"], "price": r["price"], "confidence": r["confidence"]} for r in price_results]` plus aggregate fields.
   - Always include: `new_savings`, `progress_pct`, `delta`, `product_name` (first item name or ""), `price_found` (total price), `confidence` (first item confidence or "unknown").

| MCP tool result → savings calc | Tool returns are parsed as JSON; invalid/error results are filtered by _extract_price_results |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-v98-01 | Tampering | product_name in _execute_tool | mitigate | Sanitize with `.strip()[:100]` before passing to get_price — prevents oversized MCP payloads |
| T-v98-02 | Denial of Service | _run_agentic_loop infinite loop | mitigate | MAX_TOOL_ROUNDS=5 hard cap; loop breaks when no tool_calls returned |
| T-v98-03 | Denial of Service | per-tool MCP call latency | mitigate | asyncio.wait_for timeout=12s + 2x retry with exponential backoff; timeout result logged as {"error":"timeout"} |
| T-v98-04 | Elevation of Privilege | unknown tool name in GLM response | mitigate | _execute_tool returns {"error": "unknown tool: X"} for unrecognized names; no code execution possible |
| T-v98-05 | Information Disclosure | tool result JSON parse failure | accept | Bad JSON from MCP is caught in _extract_price_results; result silently skipped; no PII at risk in price data |
